# AI-Driven 语义分词词汇映射设计

## 概述

将词汇映射从**规则驱动**改为 **AI-Driven**，让 AI 直接完成语义级分词和对应关系建立，大幅提高匹配准确率。

## 核心变化

### 1. Prompt 设计

**旧版 Prompt（规则驱动）**：
- AI 输出：英文单词 → 中文翻译
- 问题：词汇表小（~8词/段），覆盖率不足

**新版 Prompt（AI-Driven）**：
- AI 输出：中文词组 → 对应英文原文
- 优势：逐词对应，按语义划分，覆盖几乎所有文字（>90%）

```
【词汇对照表要求】
- 必须逐词对应，按语义划分词组
- 中文词组应该是译文中连续的词语
- 对应英文原文必须是输入文本中的原词/原短语
- 确保几乎所有的中文文字都在对照表中有对应（覆盖率 > 90%）

示例输出：
中文翻译：
Amical 的强大之处在于其上下文感知能力

【词汇对照表】
| 序号 | 中文词组 | 对应英文原文 |
|-----|---------|------------|
| 1 | Amical | Amical |
| 2 | 的强大之处 | The power of |
| 3 | 在于 | lies in |
| 4 | 其上下文感知能力 | context awareness |
```

### 2. 数据结构变化

**旧版结构**：
```javascript
wordMap: Map<normalizedEnglish, {originals: Set, chinese: string[]}>
// 从 wordMap 反向推导 chineseToEnglish
```

**新版结构**：
```javascript
wordMap: Map<normalizedEnglish, {originals: Set, chinese: string[]}>
chineseToEnglish: Map<chinesePhrase, englishOriginal[]>
// AI 直接提供 chineseToEnglish，无需推导
```

### 3. 分词策略变化

**旧版（算法分词）**：
1. 从 wordMap 提取中文翻译
2. 按标点分割（/、|、，等）
3. 生成子串（长词拆分为短词）
4. 算法决定分词粒度

**新版（AI 分词）**：
1. 直接使用 AI 返回的中文词组
2. AI 已按语义划分，无需再拆分
3. 仅对长词（>3字）生成子串以支持部分匹配
4. AI 决定分词粒度，更贴合语义

### 4. 解析逻辑

新增支持 AI-Driven 表格格式：
```javascript
// 检测表头：包含 "中文词组" 和 "对应英文原文"
if (line.includes('中文词组') && line.includes('对应英文原文')) {
  // 解析顺序：序号 | 中文词组 | 对应英文原文
  // 直接构建 chineseToEnglish 映射
}
```

## 优势

1. **覆盖率更高**：AI 确保几乎所有中文都有对应英文
2. **语义更准确**：AI 理解上下文，按语义划分词组
3. **减少算法猜测**：不再需要复杂的规则匹配和子串扩展
4. **双向映射直接**：中文→英文直接查表，无需遍历

## 兼容性

- 保留旧版表格格式解析（英文原文 | 中文翻译）
- 保留旧版 [词汇对应] 标记格式
- 新旧格式自动识别，无缝切换

## 调试

解析完成后会输出统计：
```
[AI Translator] AI-Driven 词汇表解析完成: 
  X 个英文词, Y 个中文词组
[AI Translator] 使用 AI-Driven 分词: Z 个词组
```

## 使用建议

1. 确保 AI 模型能理解"逐词对应"的要求
2. 对照表中虚词（的、了、在等）可以单列，也可合并到相邻词组
3. 专有名词保持独立行
4. 多词短语按语义划分，不必拘泥于单词边界
