1. 概述

1.1 项目简介

AI Web Translator 是一款基于 AI 大模型的智能网页翻译浏览器扩展，支持 Chrome 和 Safari 双平台。该扩展采用先进的语义理解技术，实现高质量的英中翻译，并提供独创的双向词汇高亮对应功能。

1.2 核心特性

  • AI 驱动翻译：基于 Kimi AI 大模型，翻译质量高，理解上下文
  • 双语对照显示：保留英文原文，中文翻译显示在原文下方
  • 双向词汇高亮：鼠标悬停词汇时，AI 自动识别对应关系，原文与译文双向高亮
  • 专业术语保留：智能识别技术词汇、专有名词，保持不翻译
  • 双平台支持：同时支持 Chrome 和 Safari 浏览器

1.3 技术栈

层级技术说明
前端框架原生 JavaScript无框架依赖，轻量级实现
样式方案原生 CSS支持深色模式

存储方案chrome.storage / browser.storage跨平台存储适配

2. 系统架构

2.1 整体架构

AI Web Translator 采用典型的浏览器扩展架构，由三个核心脚本组成：

  • Background Script（后台脚本）：处理 API 调用、跨域请求
  • Content Script（内容脚本）：注入网页，处理 DOM 操作和翻译展示
  • Popup（弹出窗口）：用户界面，配置管理

3.2.2 词汇高亮系统

词汇高亮是本扩展的独家功能，采用 AI-Driven 映射方案：

数据结构

// 英文 -> 中文 映射（原始英文，非标准化）
wordMap: Map<原始英文, {
    original: string,      // 原始英文词组
    chinese: string[]      // 对应中文词组列表
}>

// 中文 -> 英文 反向映射
chineseToEnglish: Map<中文词组, 英文原文[]>

匹配算法

        if (wordElements[i + j].dataset.word !== engWords[j]) {
            isMatch = false;
            break;
        }
    }
    if (isMatch) {
        // 高亮连续序列 wordElements[i] 到 [i + length - 1]
    }
}

5. 关键技术实现

5.1 AI-Driven 词汇映射

传统的规则映射方法依赖算法分词，准确率有限。本扩展采用 AI-Driven 方案：

提示词设计

要求 AI 返回结构化的词汇对照表：

| 序号 | 中文词组 | 对应英文原文 |
|-----|---------|------------|
| 1 | Amical | Amical |
| 2 | 的强大之处 | The power of |
| 3 | 上下文感知能力 | context awareness |

优势

  • 语义准确：AI 理解上下文，按语义划分词组
  • 覆盖率高：逐词对应，覆盖几乎所有文字
  • 无需算法：省去复杂的中文分词逻辑

5.2 并行翻译控制

采用队列 + 并发控制模式，最大化翻译效率：

const queue = [...blocks];
const executing = [];
const concurrency = 10;

while (queue.length > 0 || executing.length > 0) {
    while (executing.length < concurrency && queue.length > 0) {
        const block = queue.shift();
        const promise = processBlock(block).finally(() => {
            executing.splice(executing.indexOf(promise), 1);
        });
        executing.push(promise);
    }
    if (executing.length > 0) {
        await Promise.race(executing);
    }
}

5.3 Safari 兼容性处理

Safari 扩展与 Chrome 存在重要差异，需要特别处理：

变量声明

Safari 内容脚本共享全局命名空间，必须使用 var：

// Chrome - 正常
const isSafari = typeof browser !== 'undefined';

// Safari - 必须使用 var
var isSafari = typeof browser !== 'undefined' && browser.runtime;
var _chrome = isSafari ? browser : chrome;

存储 API 适配

var storageAPI = {
    async get(keys) {
        if (isSafari) {
            return await browser.storage.local.get(keys);
        } else {
            return new Promise((resolve) => {
                chrome.storage.sync.get(keys, resolve);
            });
        }
    }
};

7. 部署与构建

7.1 Chrome 部署

开发者模式

  1. 打开 chrome://extensions/
  2. 开启「开发者模式」
  3. 点击「加载已解压的扩展程序」
  4. 选择 chrome/ 文件夹

发布到 Web Store

  1. 打包扩展为 .zip
  2. 登录 Chrome Web Store Developer Dashboard
  3. 上传并提交审核

7.2 Safari 构建

环境要求

  • macOS 12.0+
  • Xcode 14.0+
  • Apple Developer Account（免费即可）

构建步骤

  1. 配置代码签名（Team ID）
  2. 运行 ./quick-build.sh 或 Xcode 构建
  3. 在 Safari 中启用扩展
  4. 授予网站访问权限

一键构建脚本

#!/bin/bash
# quick-build.sh

# 1. 同步代码到 Safari 项目
./scripts/sync-to-safari.sh

# 2. Xcode 构建
xcodebuild -project safari/AI\ Web\ Translator/*.xcodeproj \
    -scheme "AI Web Translator (macOS)" \
    -destination "platform=macOS" \
    build

# 3. 运行应用
open safari/AI\ Web\ Translator/build/DerivedData/...
